1. 骨架与结构 (HTML)
  文件：src/frontend/index.html

  HTML 就像是房子的结构图纸。它告诉浏览器页面上应该有什么“东西”，但不关心这些东西长什么样或者怎么动。

   * 主要元素：
       * 头部 (Header)：显示标题“空气质量实时数据流展示平台”和状态栏（连接状态、播放状态等）。
       * 控制区 (Controls)：放置了“连接”、“开始播放”、“暂停”、“重置”这几个按钮，还有一个调节速度的滑块。
       * 图表区 (Charts)：预留了三个空盒子（<div>），分别用来放 PM2.5、PM10 和 CO 的折线图。
       * 数据面板 (Data Panel)：预留了一些位置用来显示具体的数字（比如当前温度、湿度）。
   * 关键点：它引入了两个脚本文件：echarts.min.js（画图工具）和 app.js（你的业务逻辑）。

  2. 皮肤与装修 (CSS)
  文件：src/frontend/style.css

  CSS 负责美化。如果没有它，网页就是一堆黑白文字和丑陋的默认按钮。

   * 作用：它定义了背景是渐变紫色的，卡片是白色的且带阴影（看起来悬浮着），按钮有不同的颜色（蓝色、绿色、黄色），以及字体的大小和粗细。
   * 布局：它还负责排版，比如让图表并排显示，让状态栏整齐排列。

  3. 大脑与肌肉 (JavaScript)
  文件：src/frontend/app.js

  这是最关键的部分，它负责逻辑和交互。如果说 HTML 是死板的木偶，JS 就是操控木偶的线。

  你的 app.js 主要干了三件大事：

  A. 建立通信 (WebSocket)
   * 连接：当你点击“连接”按钮时，JS 会尝试连接到后端的 ws://localhost:8000/ws/stream。
   * 监听：它就像一个守门员，时刻盯着这个连接。
       * 一旦后端发来新的数据包（比如：{pm25: 30, co: 0.8, timestamp: ...}），它就会立刻抓住这个数据。

  B. 处理数据 (Data Processing)
   * 存储：它准备了几个数组（列表），用来暂存最近收到的数据。
   * 滚动更新：为了不让内存撑爆，它设置了 MAX_POINTS = 100。也就是说，列表里只保留最新的 100 个数据点，新数据进来，最老的数据就被挤出去。
   * 计算：它还内置了一个 calculateAQI 函数，用来根据收到的污染物数据实时计算 AQI（空气质量指数）。

  C. 绘制图表 (Visualization)
   * ECharts：这是一个非常强大的开源画图库（百度开发的）。
   * 渲染：每当收到新数据，JS 就会告诉 ECharts：“嘿，数据更新了，把那条折线往后画一点，把旧的擦掉一点。” 于是你就看到了实时的动态曲线。

  总结一下数据流向：

   1. 后端 (Python) 产生数据 -> 通过 WebSocket 发送。
   2. 前端 (app.js) 接收数据 -> 存入数组 -> 计算 AQI。
   3. 前端 (app.js) 调用 ECharts -> 更新网页上的图表 (index.html)。
   4. 用户 点击按钮 -> 前端发送 HTTP 请求 (/api/control/...) -> 控制后端的数据发送速度或暂停。

  这套架构非常经典，是一个标准的实时数据可视化前端实现。如果你需要修改界面文字，就动 HTML；如果想改颜色布局，就动
  CSS；如果想改数据处理逻辑或图表样式，就动 JS。
